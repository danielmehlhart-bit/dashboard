📬 Inbox-Agent — UX & User Flow

Wie Kim ihre 30-50% Email-Zeit auf 10% drückt — mit M365-Anbindung und einem Agent der mitdenkt.

1. M365-Anbindung: Wie kommt man an die Emails?

Weber Architekten hat M365 Business Basic (oder ähnlich). Die gute Nachricht: Microsoft Graph API ist in jeder M365-Lizenz enthalten — auch in den kleinen. Du brauchst kein Enterprise.

🔑 OAuth2 + Microsoft Graph API

So funktioniert's: Du registrierst Architekt Connect als App in Microsofts Azure AD (Entra ID). Kim klickt einmal "Erlauben" — fertig. Ab dann kann dein Inbox-Agent ihre Emails lesen.

  • Scope: Mail.Read — nur lesen, nie senden oder löschen
  • Kosten: 0€ — Graph API ist in jeder M365-Lizenz inkludiert
  • DSGVO: Daten bleiben bei Microsoft, dein Agent liest nur Metadaten + Body on-demand
  • Kein Admin nötig: Bei kleinen Büros ohne IT-Admin kann jeder User selbst consenten
Wichtig — Privacy by Design: Der Inbox-Agent liest nicht ALLE Emails. Er liest nur Emails die mit Projekten matchen (Absender, Betreff, Keywords). Private Emails, Newsletter, HR-Kram = unsichtbar. Das muss Kim sehen und verstehen, sonst fehlt das Vertrauen.

5. Proaktive Intelligenz — Der Agent denkt mit

Der Inbox-Agent ist nicht nur ein Email-Sortierer. Er lernt Muster und wird proaktiv:

  • Überfälligkeits-Tracking: "Statiker Müller sollte den Bewehrungsplan bis Montag liefern (laut Email vom 20.02.). Heute ist Donnerstag — noch keine Mail. Soll ich eine Erinnerungs-Email vorschlagen?"
  • Duplikat-Erkennung: "Familie Schmidt hat den Küchenwunsch schon am 15.01. erwähnt — Task existiert bereits. Soll ich den Task updaten statt neu anlegen?"
  • Eskalation: "3 unbeantwortete Emails vom Bauamt in 2 Wochen → Priorität hochgestuft."
  • Digest: Freitag 16:00 automatischer Wochenrückblick: "Diese Woche: 12 Projekt-Emails → 8 Tasks erstellt → 5 erledigt → 3 offen."

6. Technische Architektur — Einfach gehalten

Kim's Outlook (M365) │ ▼ Microsoft Graph API (OAuth2, Mail.Read) │ ▼ Supabase Edge Function "inbox-agent" ├── Polling alle 5 Min (oder Graph Webhook für Realtime) ├── Email lesen → Projekt-Matching (Absender-Map + NLP) ├── LLM-Call: Zusammenfassung + Task-Extraktion └── Ergebnis → Supabase DB "inbox_items" Tabelle │ ▼ Architekt Connect Frontend ├── Inbox-View (Realtime via Supabase Subscriptions) ├── Task-Erstellung (1-Click aus Suggestion) └── Feedback-Loop (Kim korrigiert → Agent lernt)

Warum Supabase Edge Function?

  • Läuft serverless, keine Infrastruktur-Wartung
  • Direct DB-Access (kein extra API Layer)
  • Deno Runtime = TypeScript, passt zu deinem Stack
  • Cron-Trigger: Supabase pg_cron → Edge Function alle 5 Min
  • Kosten: ~$0.01-0.05 pro 100 Emails (LLM-Token für Summaries)

Graph API — Was du brauchst

  • App Registration: Azure Portal → App registrieren (kostenlos, auch ohne Azure-Abo)
  • Permissions: Mail.Read (delegated) — User-Consent reicht, kein Admin
  • Token Storage: Refresh Token in Supabase (encrypted), auto-refresh
  • Webhook (optional): subscriptions API für Push statt Polling — spart Calls
  • Rate Limit: 10.000 Requests/10 Min — mehr als genug für kleine Büros

7. Datenschutz — Vertrauen ist alles

DSGVO-Strategie: Minimal-Prinzip
  • Kein Email-Inhalt speichern. Der Agent liest die Email, extrahiert Summary + Tasks, und speichert nur die Extraktion. Der volle Email-Body bleibt bei Microsoft.
  • Projekt-Filter first. Nur Emails von bekannten Projekt-Kontakten werden analysiert. Alles andere = unsichtbar.
  • Transparenz-Log. Kim sieht jederzeit: "Der Agent hat heute 47 Emails gesehen, 12 analysiert, 35 ignoriert." Kein Blackbox-Gefühl.
  • Jederzeit trennbar. Ein Klick "Email trennen" → OAuth revoken → fertig. Keine Datenleichen.
  • AVV mit Kim's Büro. Auftragsverarbeitungsvertrag — muss sein, ist bei M365-Integration Standard. Template reicht.

8. Pricing-Implikation

Der Inbox-Agent ist ein Premium-Feature. Das ist dein Upsell von der Basis-PM-Lizenz:

  • Basis (79-99€/Monat): PM + Tasks + Bautagebuch + Voice — das Human Layer
  • Pro (149-199€/Monat): + Inbox-Agent + Controller-Agent + AI-Summaries — das Agent Layer
  • Rechnung für Kim: "Der Inbox-Agent spart dir 1-2h pro Tag. Das sind ~200-400€/Monat an Arbeitszeit. Du zahlst 100€ mehr. ROI ab Tag 1."
