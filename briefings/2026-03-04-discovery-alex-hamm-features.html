🚀 Feature #1: Inbox Agent (Email-Triage)

CORE FEATURE P0

Problem Statement

Alex bekommt 250-300 Emails/Tag. Jede Email ist eine Projekt-Info oder Aufgabe, aber isoliert in Outlook. Es gibt keine automatische Zuordnung zu Projekten, keine Priorisierung, und wenn jemand vergisst CC zu setzen, geht Information verloren. Nach 2 Tagen Abwesenheit: 500-600 Emails Backlog.

Lösung

KI-Agent der Emails liest, zu Projekten zuordnet, zusammenfasst, und 1-Click Tasks generiert. Alex sieht morgens nicht 300 Emails sondern 10-15 projektrelevante, vorverdaute Items.

User Flow

📧 Email kommt rein
→
🤖 Agent liest & klassifiziert
→
📁 Projekt-Zuordnung
→
📝 Summary + Task-Vorschlag
→
✅ 1-Click Task erstellen

Acceptance Criteria

  • Email-Provider anbinden (Gmail MVP, M365 Produktion) via OAuth read-only
  • Emails automatisch Projekten zuordnen (Absender-Map + Betreff-Keywords)
  • KI-Summary pro Email (1-2 Sätze, deutsch)
  • Task-Vorschlag extrahieren (Titel, Assignee, Deadline, Priorität)
  • 1-Click "Task erstellen" → Task landet im richtigen Projekt
  • "Ignorieren" → Email wird nicht mehr angezeigt
  • Transparenz-Log: "47 Emails gesehen, 12 analysiert, 35 ignoriert"
  • Kein Email-Body gespeichert — nur Summary + Metadaten (DSGVO)
  • Jederzeit trennbar (OAuth revoke)

Alex-spezifische Anforderungen

  • Team-Emails: Nicht nur Alex — alle Teammitglieder können Emails forwarden
  • Antwortvorschläge: "Auch schon Antwortvorschläge generieren" (Alex bestätigte Interesse)
  • Eskalation: Unbeantwortete Emails nach X Tagen hochstufen
  • Wochenrückblick: Freitag 16:00 — "12 Emails → 8 Tasks → 5 erledigt → 3 offen"

Tech-Architektur

KomponenteLösung

AuthOAuth2 read-only (gmail.readonly / Mail.Read)
BackendSupabase Edge Function "inbox-agent" (Cron alle 5 Min)

FrontendInbox-View mit Cards + 1-Click Actions

🚀 Feature #2: Checklisten-Automatisierung (Projekt-Kickstart)

CORE FEATURE P0

Problem Statement

Jedes neue Projekt (Neubau, Nutzungsänderung, Sanierung, etc.) hat die gleichen Standardschritte. Alex erklärt sie jedes Mal neu, Mitarbeiter vergessen sie nach 3 Monaten Pause. Es gibt keine Templates, kein institutionelles Wissen im System. 80% der Anfragen werden abgelehnt — nicht wegen fehlendem Können, sondern fehlender Kapazität durch repetitive Prozesse.

Lösung

Projekttyp auswählen → System generiert automatisch alle Standard-Tasks mit Checklisten, weist zu, setzt Deadlines. Bauherr bekommt seine Todos automatisch. Kein manuelles Erklären, kein Vergessen.

User Flow — Intern (Architekt)

📋 Neues Projekt anlegen
→
🏗️ Projekttyp wählen (Neubau/Sanierung/...)
→
⚡ System generiert Tasks + Checkliste
→
👤 Tasks zuweisen + Deadlines
→
✅ Team arbeitet Checkliste ab

User Flow — Extern (Bauherr)

✍️ Angebot unterschrieben
→
📧 Bauherr bekommt Login
→
📝 Checkliste mit ersten Todos
→
✅ Bauherr arbeitet ab (vor Ersttermin)
→
📊 Architekt sieht Fortschritt

Acceptance Criteria

  • Projekttyp-Templates definierbar (Neubau, Nutzungsänderung, Sanierung, Umbau, Gewerbe)
  • Jedes Template hat Standard-Tasks mit: Titel, Beschreibung, Checkliste, Default-Deadline (relativ), Default-Assignee-Rolle
  • Bei Projekt-Erstellung: Template auswählen → Tasks werden automatisch generiert
  • Bauherren-Checkliste: Subset der Tasks markiert als "Bauherr-relevant"
  • Bauherr bekommt Email mit Login-Link + seine Checkliste
  • Fortschritts-Tracking: Welche Items hat der Bauherr erledigt? (Sichtbar für Architekt)
  • Admin kann Templates verwalten (CRUD)
  • Info-Buttons an Tasks: "Was ist eine Genehmigungsplanung?" → Erklärungs-Popup (Wiki-light)
