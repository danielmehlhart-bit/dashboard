📱 Telegram Topics + Multi-Agent
Research Briefing

Wie Forum-Gruppen als Projekt-Workspace funktionieren — Theorie, Pitfalls & Empfehlung für Daniels Setup.
🔵 Telegram 🤖 OpenClaw 🏗 Multi-Agent 📋 Setup-Guide
01 — Grundlagen

💬 Was sind Telegram Forum Topics?

  • Telegram Gruppen können seit 2022 einen „Forum-Modus" aktivieren
  • Statt einem Chat gibt es mehrere Threads / Topics — wie Slack Channels innerhalb einer Gruppe
  • Jedes Topic hat einen eigenen Nachrichtenverlauf, komplett isoliert
  • Perfekt für: Projekt-Isolation, Multi-Agent Setups, thematische Trennung von Kontexten
Kernidee: Eine Telegram-Gruppe wird zum persönlichen AI-Cockpit — jedes Topic ist ein eigener Arbeitsbereich mit eigenem Agenten.
02 — OpenClaw Integration

⚙️ Wie OpenClaw Topics unterstützt

  • Jedes Topic bekommt eine eigene Session: :topic:<threadId>
  • Eigener Kontext, eigene History, eigene Compaction — vollständig isoliert
  • Topics erben Gruppen-Settings, können aber individuell überschrieben werden

Per-Topic Konfigurationsoptionen

channels.telegram.groups.<chatId>.topics.<threadId>:
  agentId:       "hatti"         // eigener Agent pro Topic
  systemPrompt:  "..."           // spezifischer Kontext
  skills:        ["strava"]      // nur relevante Skills
  requireMention: false         // ob Bot erwähnt werden muss
  allowFrom:     ["daniel"]      // Zugriffskontrolle
  groupPolicy:   "allowlist"     // open / allowlist / disabled
03 — Best Practices

🛠 Setup-Schritte

  1. Telegram Forum-Gruppe erstellen (nur du + Bot, Supergroup)
  2. Topics aktivieren in den Gruppeneinstellungen
  3. Thread-IDs erfassen (aus Message-Links oder API)
  4. Bei BotFather: /setprivacy → disable — damit Bot alle Nachrichten sieht, nicht nur Mentions
  5. Bot zur Gruppe hinzufügen (als Admin mit entsprechenden Rechten)
  6. openclaw.json anpassen: Gruppe + Topics konfigurieren

🤖 Single-Bot Unser Fall

1 Bot, mehrere Topics, verschiedene System Prompts

  • + Einfach, weniger Config
  • + Schnell aufzusetzen
  • – Gleicher Agent-Context
  • – Nur System Prompt unterscheidet sich

🤖🤖 Multi-Bot Advanced

Mehrere Bots, jeder im eigenen Topic, eigene Agents

  • + Komplett isolierte Workspaces
  • + Eigene Memory & Skills pro Bot
  • – Mehrere BotFather-Bots nötig
  • – Deutlich mehr Konfiguration

🎯 Dan Malones „Mission Control" Pattern

  • 4 spezialisierte Agents in einer einzigen Forum-Gruppe
  • Jeder Agent hat: eigenen Workspace, SOUL.md, Memory, Skills
  • Inter-Agent Kommunikation via sessions_send (z.B. Goaly fragt Homey)
  • Fin (Buchhaltung) · Goaly (Tasks) · Homey (Smart Home)
  • 3 Modi: Deep Work (Terminal) · On-the-go (Telegram) · Always-on (Crons)

„Die Lücke war nicht Coordination UI, sondern Persistenz + Mobile + Cross-Agent Collaboration."

— Dan Malone, dan-malone.com, 18.02.2026
04 — Häufige Fehler & Pitfalls

🔧 Config-Fehler

  • ❌ groups Config global statt per-account → alle Bots antworten überall
  • ❌ Ungültige Topic-Keys (z.B. label) → nur Schema-konforme Keys
  • ❌ Privacy Mode aktiviert gelassen → Bot sieht keine unerwähnten Nachrichten
  • ❌ Bot nicht neu hinzugefügt nach Privacy-Änderung → entfernen & neu einladen

🧠 Kontext-Probleme

  • ❌ Zu großer System Prompt → Token-Verschwendung bei jedem Turn
  • ❌ Keine klare Trennung → Topics werden zu „zweitem General Chat"
  • ❌ Memory-Leakage: Agent referenziert anderen Topic-Kontext
  • ✅ Fix: Pro Topic kurzer, spezifischer System Prompt mit klarem Scope

🏛 Architektur-Fehler

  • ❌ Zu viele Topics → Fragmentierung, man verliert den Überblick
  • ❌ Kein „General" Topic → alles muss kategorisiert werden
  • ✅ Empfehlung: 3–5 Topics maximal + 1 General für Unkategorisiertes

📱 Telegram-spezifisch

  • ❌ Reactions haben keine Thread-ID → gehen an General Topic
  • ❌ General Topic (threadId=1) hat Sonderbehandlung: message_thread_id wird nicht gesendet
  • ❌ Bot kann nicht selbst Topics erstellen → braucht Admin-Rechte
05 — Empfehlung für Daniels Setup

✅ Option A: Minimal — Single Bot (Empfohlen als Start)

  • @hatti_berg_bot in einer Forum-Gruppe mit 4–5 Topics
  • Pro Topic: eigener System Prompt mit Repo-Pfad + Projekt-Kontext
  • 1:1 DM Chat bleibt als Hauptkanal erhalten
  • Einfach, sofort umsetzbar, ausbaubar

🔮 Option B: Full Multi-Agent (für später)

  • Separater Bot pro Projekt (z.B. @modulai_bot, @bpp_bot)
  • Komplett isolierte Workspaces und Memory pro Bot
  • Inter-Agent Kommunikation via sessions_send
  • → Wenn mehr Projekte / Kunden, dann diesen Weg gehen
